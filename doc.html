Problem Statement

By using Dynamic Programming Algorithm implement an algorithm to predict the secondary structure of a RNA and return all the base-pairs.

Dynamic Programming solution

Let \(dp_{l, r }\) denote the cardinality of the maximum matching in \(s_l s_{l+1} \ldots s_r\) of the sequence \(S\). Then it is easy to notice the following recurrence.

\[ dp_{ l, r } = \max \begin{cases} \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } 0 & r - l \le \texttt{4} \\ \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } \text{ } dp_{ l, r-1 } & r - l \gt \texttt{4} \\ \underset{l \le m \le r}{\max} \text{ } \text{ } dp_{ l, m-1 } + 1 + dp_{ m+1, r-1} & b_m \text{ and } b_r \text{ are complementary bases} \end{cases} \]

The required value is \(dp_{ 1, n }\) â€” the maximum matching in the sequence \(S\).

Timing Analysis

The window size takes \(\mathcal{O}(n)\) time to compute since we increase the value of k from 5 to n as the outer loop and there are \(\mathcal{O}(n^2)\) states. Hence the time complexity of computing all the dp which is our opt[][] states is \(\mathcal{O}(n^3)\).

Also, since there are \(\mathcal{O}(n^2)\) dp states which must be stored in memory, the space complexity is \(\mathcal{O}(n^2)\).

Design Problem

The design problem in this algorithm is that the time complexity is high. The algorithm has a time complexity of \(\mathcal{O}(n^3)\), where \(n\) is the length of the RNA sequence. This means that as the length of the sequence increases, the time taken to compute the maximum matching also increases significantly. This can be a performance bottleneck, especially for large RNA sequences. It also restricts the length of the RNA sequence that can be processed efficiently.
